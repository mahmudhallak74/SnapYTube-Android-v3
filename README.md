# 📱 SnapYTube Ultimate — تطبيق أندرويد مستقل (Kivy)

هاد المشروع تطبيق أندرويد **حقيقي ومستقل** — بايثون (Flask + yt-dlp) بيشتغل جوا التطبيق نفسه
كـ thread بالخلفية، وواجهة `index.html` نفسها بتنعرض بـ WebView أصلي (native) جوا الشاشة.
**ما في حاجة لـ Termux إطلاقاً بعد التنصيب.**

## ⚠️ اقرأ هاد قبل ما تبلش

بناء تطبيق بايثون لأندرويد (Kivy/buildozer) بيحتاج ترجمة كل مكتبة لكود أندرويد أصلي — عملية
معقدة ومعروفة إنها بتفشل لأسباب متنوعة (نسخة SDK، تراخيص، توافق مكتبات...). ما عندي بيئة أختبر
فيها البناء الفعلي قبل ما توصلك الملفات، فمش مضمون ينجح من أول محاولة ١٠٠٪.
**لو فشل**: انسخ نص رسالة الخطأ كاملة (مو صورة) من صفحة الـ Actions وابعتها، هيك بصلحها بدقة
أسرع بكثير من التخمين على صورة.

## ✨ ميزات التطبيق

### 📥 التحميل
- تحميل فيديوهات من يوتيوب، تيك توك، انستغرام، فيسبوك، تويتر/X، فيميو.
- كشف تلقائي للمنصة من الرابط.
- شريط تقدم حي (سرعة، نسبة الإنجاز، الوقت المتبقي) عبر تحديث لحظي (SSE).
- حماية من إغراق السيرفر بالطلبات (Rate limiting).

### 📂 الملفات المحفوظة
- تبويبات منفصلة: فيديو / صور / صوت.
- عرض مكان الحفظ الفعلي على الجهاز.
- حفظ/مشاركة أي ملف مباشرة من الواجهة.

### 📱 التطبيق نفسه
- **مستقل بالكامل** — سيرفر Flask و yt-dlp يشتغلوا جوا التطبيق نفسه (بدون Termux بعد التنصيب).
- شاشة تحميل حقيقية عند الفتح (فحص جاهزية السيرفر فعلياً، مش انتظار وقت ثابت).
- واجهة عربية كاملة (نفس تصميم النسخة الأصلية).
- يعمل من أندرويد 5.0 فما فوق (`minapi=21`)، معماريتين `arm64-v8a` + `armeabi-v7a` (يغطي الأجهزة الحديثة والقديمة/الرخيصة سوا).
- تخزين بمجلد التطبيق الخاص (بدون حاجة لصلاحيات وصول كامل للملفات).
- كل جهاز ينصّبه يشتغل بشكل مستقل تماماً — بدون اعتماد على أي سيرفر مركزي.

### ⚠️ قيود حالية (تحسينات مستقبلية ممكنة)
- بدون ffmpeg مدمج → الجودات العالية جداً من يوتيوب (1080p+ المنفصلة) ممكن ما تنزل، بس أغلب المنصات (تيك توك/انستغرام/فيسبوك) وفيديوهات يوتيوب حتى 720p بتشتغل عادي.
- تحديثات yt-dlp تصير عن طريق نسخة APK جديدة (مو تحديث حي متل نسخة Termux).
- APK الحالي **debug build غير موقّع بمفتاح release** — كافي للتنصيب والاستخدام الشخصي، بس لو بدك تنشره على متجر أو توزّعه على نطاق واسع، لازم `buildozer android release` + توقيع بمفتاح release خاص فيك.

### 🔧 تحديثات هالنسخة
- ✅ إضافة معمارية `armeabi-v7a` جنب `arm64-v8a` — قبل هيك أي جهاز 32-بت (قديم/رخيص) ما كان يقدر يثبت التطبيق إطلاقاً.
- ✅ حل جزئي/محسّن لمشكلة "تيك توك status code 0": headers تشبه تطبيق تيك توك الحقيقي + `extractor_args` مخصص + محاولة ثانية تلقائية بإعدادات مبسطة لو فشلت الأولى. **ملاحظة**: تيك توك بيغيّر حمايته بشكل متكرر، فمافي حل نهائي 100% دائم — بس هاد أثبت طريقة معروفة بمجتمع yt-dlp بدون الاعتماد على `curl_cffi` (يلي بيصعب/بيكسر بناء python-for-android).

## 🗂️ بنية المشروع
```
main.py              → نقطة الدخول (Kivy App) — بيشغّل السيرفر ويفتح الـ WebView
server.py             → Flask app (نفس الـ API endpoints تبع النسخة يلي عندك)
mobile_downloader.py  → محرك yt-dlp مبسّط لجهاز واحد
index.html            → نفس الواجهة (بدون أي تعديل)
buildozer.spec        → إعدادات البناء (الصلاحيات، المكتبات، اسم الحزمة...)
.github/workflows/build.yml → بناء أوتوماتيكي عبر GitHub Actions (باستخدام buildozer-action الجاهزة)
```

## ⚠️ نقطة مهمة جداً: ffmpeg

`yt-dlp` بحاجة `ffmpeg` عشان يدمج الفيديو والصوت لما بيكونوا منفصلين (خصوصاً يوتيوب بجودة عالية).
مشكلة `ffmpeg` إنه:
- ما في recipe رسمي جاهز له بـ `python-for-android` تحطه بالـ `requirements`.
- لازم تجيب **binary ثابت مبني لـ arm64-v8a** (متل نسخة johnvansickle الثابتة أو بناء termux-packages)
  وتحطه بمسار `assets/bin/ffmpeg` قبل البناء، وبعدين `main.py`/`server.py` بينسخوه أول تشغيل
  لمجلد التطبيق ويعطيه صلاحية تنفيذ (`chmod +x`).

**لحد ما تجهز ffmpeg**: الكود حالياً شغّال بوضع "بدون دمج" —
بياخد صيغة `mp4` جاهزة مدموجة من المصدر مباشرة (`format: best[ext=mp4]/best`)، يعني:
- ✅ يوتيوب لغاية 720p عادةً بيشتغل تمام (في صيغة mp4 مدموجة جاهزة).
- ⚠️ الجودات الأعلى (1080p+) أحياناً بتكون منفصلة (فيديو لحاله وصوت لحاله) وما رح تنزل بدون ffmpeg.
- ✅ تيك توك / انستغرام / فيسبوك عادة ما بتحتاج دمج أصلاً.

إذا بدك الجودات العالية شغالة 100%، خبرني وبساعدك تجهز ffmpeg binary وتدمجه بالمشروع.

## 💾 مكان الحفظ

الملفات بتنحفظ بمجلد التطبيق الخاص (`app_storage_path()/downloads`)، يعني:
- ما بيحتاج صلاحية "الوصول لكل الملفات" (Android 10+ scoped storage).
- بتلاقيه عن طريق تطبيق "الملفات" تحت: `Android/data/com.mahmud.snapytube/files/downloads`
- الواجهة (تبويب "الملفات المحفوظة") بتعرضلك المسار الكامل مباشرة.

## 🚀 خطوات البناء (عن طريق GitHub، بدون كمبيوتر)

1. اعمل push للمجلد كامل لمستودع GitHub جديد (أو استخدم مستودع SnapYTube الحالي بفرع جديد).
2. GitHub Actions بيبلش أوتوماتيك (أو شغّله يدوياً من تبويب Actions → Run workflow).
3. بعد ما يخلص (٥-١٥ دقيقة أول مرة)، روح لـ Artifacts وحمّل `snapytube-apk-unsigned`.
4. فك الضغط، جوا رح تلاقي APK غير موقّع.
5. وقّعه بنفس طريقتك المعتادة:
   ```
   zipalign -v 4 snapytube-debug.apk snapytube-aligned.apk
   apksigner sign --ks your-key.keystore snapytube-aligned.apk
   ```
6. نصّب الـ APK عادي.

> ✅ ما في داعي لـ `apktool` هون — هاد مشروع بايثون كامل بيتبنى من الصفر عبر `buildozer`،
> مش تعديل APK جاهز. `apktool` بينفع لما تفكك تطبيق موجود، مو لما تبني مشروعك من الكود.

## 🔄 التحديثات

بما إنه صار تطبيق مستقل، ما في تحديث حي لـ `yt-dlp` متل نسخة Termux (ما في وصول لـ pip بعد التنصيب).
كل ما بدك نسخة yt-dlp أحدث → تعدّل رقمها بـ `requirements` بملف `buildozer.spec`، تعمل push،
وبتطلعلك نسخة APK جديدة عن طريق GitHub Actions. لو حابب، فيني أضيفلك خطوة بالـ workflow
تتحقق تلقائياً من أحدث نسخة yt-dlp قبل كل بناء.
